💡 專案核心設計理念與開發心得

1. 程式碼參考來源

本遊戲電腦端的底層核心演算法與視窗架構，參考自知乎黑馬程式設計師的經典教學文章：《石頭迷陣》手把手開發教學。

2. AI 輔助跨語言轉換歷程

由於標準的 Java 語言（Swing/JFrame 架構）無法直接在現代網頁瀏覽器中運行（舊式的 Java Applet 技術已被安全淘汰），為了能在個人網頁上完美呈現專案成果，本專案使用 AI 工具進行輔助技術轉換。AI 協助將原生的 Java 物件導向邏輯，無縫重構成網頁專用的 HTML5、CSS3 與 JavaScript 架構，在保留核心資料結構的同時，大幅提升了跨平台的便利性與視覺美感。

3. 課堂核心 Java 觀念應用

不論是原生的 Java 桌面端還是移植後的網頁端，本專案皆嚴格實踐了課堂上所學到的三大核心資訊觀念：

  • 視窗繼承（Inheritance）： 原生專案透過 extends JFrame 繼承 Java 內建視窗，免去重新發明輪子，直接在外殼上堆疊遊戲邏輯。
  • 虛擬棋盤對映（2D Array）： 核心資料結構採用 4x4 的**「二維陣列」**，將畫面上的視覺方塊與幕後的矩形矩陣做一對一的資料綁定。
  • 數據與畫面的分離（Data-Driven UI）： 手控制的是二維陣列的數字交換（資料層），眼看到的是經由 repaint() 刷新後的方塊（畫面層），實現資料驅動視覺的進階架構。

4. 「死局檢查器」的想法由來

在最初的版本中，我們發現隨機打亂方塊後，遊戲有高達 50% 的機率會陷入永遠無法解開的「死局」（例如最後只剩 14 和 15 順序顛倒）。

為了解決這個致命痛點，我們引入了數學上的「逆序數（Inversion Count）」定理。在 4x4 的拼圖遊戲中，將所有數字排成一列後，計算前面數字大於後面數字的組合總數。由於最終期望目標是將空格留在最右下角（即第 3 列，索引為偶數），因此逆序數的奇偶性必須與空格所在的列索引奇偶性完全一致。本專案以此原理建構了**「死局檢查器」**，在洗牌瞬間進行數學審查，若發現不符（死局），則在 0.001 秒內自動重新洗牌，確保玩家開局 100% 能夠順利破關！

package stonegame;

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;
import java.util.Random;

// 應用課堂觀念：繼承 JFrame，並實作事件監聽介面
public class StoneGame extends JFrame implements KeyListener, ActionListener {
    // 應用課堂觀念：使用 4x4 二維陣列記錄棋盤數據
    private int[][] data = new int[4][4]; 
    private int row0, col0;               // 記錄空格(0)的位置
    private int step = 0;                  // 計步器

    public StoneGame() {
        this.setSize(514, 595);
        this.setTitle("石頭迷陣單機版 v1.0");
        this.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        this.setLocationRelativeTo(null);
        this.setLayout(null); // 取消預設佈局
        this.addKeyListener(this); // 註冊鍵盤監聽
        
        initData();  // 打亂資料邏輯
        initImage(); // 畫面渲染 (資料與畫面分離觀念)
        this.setVisible(true);
    }
    // ... 後續包含鍵盤 W/A/S/D 控制與陣列元素互換(Swap)邏輯
}
