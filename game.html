💡 專案核心設計觀念

  1. 視窗繼承（簡化介面開發）： 專案透過繼承直接借用了內建的視窗元件。這讓我們不需要從零開發「如何產生視窗、如何點擊關閉」等基礎外殼，能夠將焦點完全集中在遊戲本身的互動邏輯上。
  2. 虛擬棋盤對映（二維陣列的應用）： 遊戲的靈魂是一組 4×4 的二維陣列網格。這就像是在幕後建立了一個隱形的虛擬棋盤，用來精準記錄 1 到 15 的數字與空格的相對位置。
  3. 動態事件監聽（實現人機互動）： 為了讓程式能夠即時接收玩家的指令，專案加入了監聽機制。當玩家按下鍵盤或選單時，程式能瞬間捕捉到動作，並立刻做出方塊移動或重新開始的反應。
  4. 數據與畫面的分離（資料驅動視覺）： 這是遊戲最核心的運作邏輯：「手控制的是數字，眼看到的是畫面」。當玩家移動方塊時，程式本質上只是在幕後交換資料，資料改變後，才通知畫筆在前端重新整理。

    private int step = 0;                  // 計步器

    public StoneGame() {
        this.setSize(514, 595);
        this.setTitle("石頭迷陣單機版 v1.0");
        this.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        this.setLocationRelativeTo(null);
        this.setLayout(null);
        this.addKeyListener(this);
        
        initData();  // 打亂資料邏輯
        initImage(); // 畫面渲染
        this.setVisible(true);
    }

}
